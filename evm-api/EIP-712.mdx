---

Orderly Network utilizes the following domain and types for the registration, add access key, withdrawal and settle PnL processes.

### Domain
Note that chainId will be the id of the chain connected, which must be supported by Orderly Network.

```json
const domain = {
    name: 'Orderly',
    version: '1',
    chainId: chainId,
    verifyingContract: '0xCcCCccccCCCCcCCCCCCcCcCccCcCCCcCcccccccC',
}
```

### Types

```json
const types = {    
    "EIP712Domain": definedTypes["EIP712Domain"],
    "Registration": definedTypes["Registration"],
    "Withdraw": definedTypes["Withdraw"],
    "AddOrderlyKey": definedTypes["AddOrderlyKey"],
}
```

### Type Definition

```json
const definedTypes = {
    "EIP712Domain": [
        { name: 'name', type: 'string' },

    "Registration": [
        {name: "brokerId", type: "string"},

        {name: "timestamp", type: "uint64"},
        {name: "registrationNonce", type: "uint256"}
        ],
    "AddOrderlyKey": [
        {name: "brokerId", type: "string"},
        {name: "chainId", type: "uint256"},
        {name: "orderlyKey", type: "string"},
        {name: "scope", type: "string"},
        {name: "timestamp", type: "uint64"},
        {name: "expiration", type: "uint64"},
    ],
    "Withdraw": [
        {name: "brokerId", type: "string"},
        {name: "chainId", type: "uint256"},
        {name: "receiver", type: "address"},
        {name: "token", type: "string"},

        {name: "brokerId", type: "string"},

    ],
}
```
